CSS typography

Notes from various sources

Importing fonts

It used to be a pain in the arse but now we use @fontface and specify a url to the font file the browser can download then use .selector {font-family: 'name-of-font'} to use it on certain elements.

Another option is to use an online font service such as google fonts or adobe typekit. These are a breeze to use as they host, serve and optiise the font for you(me). For the purpose of learning this tutorial I'll use Rubik from Google fonts.

Font sizing

CSS has 15 different units for font sizing. They can be grouped in 2 categoiries:
  1. Absolute such as px
  2. Relative such as em

Relative sizes take their cue from the parent element. So if a paragraph is set to 2em and its parent element, let's say a div, is set to 16px, that 1em in the paragraph translates to 32px.

Vertical spacing

Usually achieved via the line-height property.

Is a way to control the distance between lines of text. Usually the browser defaults to 1.2, but that is too tight. For long form text something closer to 1.5 works better.

This combined with margins, padding etc. forms (vertical) rythm.

Horizontal spacing

Tracking is the space between letters of a word. It's controlled by the letter-spacing property. Ems are the preferred units of measurement.

Line width - best practice for long form copy is to have between 60 and 70 characters per line. Use the ch unit. It is a relative unit define by the lentgh of tha character 0 (zero).

"Because narrow characters like l and i are relatively frequent, setting the width of a text container to something like 50ch should result in lines that are 60-70 character long."

Antialiasing

Rendering gray pixels to make fonts appear smoother.

-webkit-font-smoothing - supported only on webkit browser but firefox and chrome must have an equivalent.
